# Code of Conduct

中文摘要：请保持专业、克制和可协作。争论可以尖锐，但必须围绕事实、实现和用户影响，不能针对人。

## Our Standard

Participants in this repository are expected to:

- communicate respectfully
- keep technical disagreement concrete and evidence-based
- assume good intent until behavior shows otherwise
- protect private information, credentials, and security-sensitive details
- keep issue and PR discussion focused on the work

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment, insults, or personal attacks
- discrimination or hateful language
- repeated bad-faith disruption of discussion or review
- publishing private data, credentials, or exploit details irresponsibly
- spamming, derailing threads, or intentionally wasting reviewer time

## Scope

This code of conduct applies to:

- issues
- pull requests
- discussions
- review comments
- other repository-linked collaboration channels

## Enforcement

Maintainers may:

- ask for changes in tone or behavior
- hide or remove comments
- lock threads
- reject contributions
- restrict further participation for repeated or severe violations

## Reporting

Use GitHub moderation tools when appropriate. For security-sensitive conduct or content, prefer a private reporting path and avoid posting exploit details publicly.

## Project Norm

This project values direct technical feedback. Being blunt about code, tests, or design is acceptable; being disrespectful to people is not.
